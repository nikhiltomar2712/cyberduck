# Phase 3: Web & Network Security (6–8 weeks)

> **Ethics & Authorization**: Practice only in safe local labs.

## Learning objectives
- Identify and mitigate common web vulnerabilities.
- Apply defensive design and secure configuration patterns.
- Understand network traffic at a defender-friendly level.
- Convert observations into detection and hardening recommendations.

## Key topics (expanded)
- **Web application security**
  - OWASP Top 10 (defender view)
  - authentication/session management
  - authorization and access control
  - input validation and output encoding
  - security headers and cookie flags
  - dependency and configuration security
- **Networking & traffic analysis**
  - DNS and service exposure concepts
  - HTTP/HTTPS traffic signals (defender perspective)
  - logging strategy for web services

## Weekly goals & mini-projects

### Week 1: Web security basics → secure checklist
- Mini-project: build a “web app review” checklist.

### Week 2: Authentication & sessions hardening
- Mini-project: propose hardening steps for session controls (secure cookies, logout behavior, rate limiting concepts).

### Week 3: Authorization & broken access control thinking
- Mini-project: threat model an access-control flow and propose verification tests.

### Week 4: Security headers + transport
- Mini-project: create a security header baseline and explain what each header protects.

### Week 5–6: Safe lab review with vulnerable apps
- Use local safe labs (no real exploitation against external targets): DVWA / Juice Shop.
- Mini-project: produce write-ups focusing on **findings → impact → mitigations**.

### Week 7–8 (optional): Network monitoring signals
- Mini-project: define what logs/metrics you’d monitor for suspicious patterns.

## Success checkpoints
- You can map a web risk to a defensive mitigation and verification step.
- You can produce a short, evidence-based web security write-up.
