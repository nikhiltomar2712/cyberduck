# Glossary & Terminology

> **Ethics & Authorization**: Use this material only on authorized defensive learning work.

This glossary is intentionally short and beginner-friendly. As you progress through phases, expand it with your own notes.

## Core security terms

| Term | Meaning |
| --- | --- |
| Asset | Something valuable (data, accounts, systems, APIs, services). |
| Threat | A possible cause of harm. |
| Vulnerability | A weakness that could be exploited. |
| Risk | Likelihood × impact; what you need to manage. |
| Control | A safeguard that reduces risk. |
| Exposure | Where an asset can be reached (network, auth paths, APIs). |
| Incident | An event that harms or threatens to harm security goals. |
| Triage | Prioritizing and deciding what to do first during an incident. |
| Evidence | Records that support what happened (logs, timestamps, artifacts). |
| Verification | Steps to confirm whether a mitigation works. |

## Threat modeling terms

| Term | Meaning |
| --- | --- |
| Trust boundary | A line where trust changes (e.g., user → server). |
| Data flow | How information moves through a system. |
| STRIDE | A structured way to think about threat categories. |

## SOC terms

| Term | Meaning |
| --- | --- |
| Alert | A signal that might represent malicious or risky behavior. |
| Detection hypothesis | What you think the alert might mean. |
| Investigation | Defensive analysis to confirm/deny the hypothesis. |
| Response action | A decision or step to reduce damage and recover. |
