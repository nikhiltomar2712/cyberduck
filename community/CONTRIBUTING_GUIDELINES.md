# Community Contribution Guidelines

> **Ethics & Authorization**: All contributions must support legal/authorized defensive cybersecurity education.

## What you can contribute
- Beginner-friendly security education
- Defensive checklists and hardening guides
- Lab guides that use safe local environments only
- Templates for reports, timelines, and audits
- Documentation improvements (clarity, structure, links)
- Code contributions to the Cyberduck Python helpers (with tests)

## What you must not contribute
- Instructions to attack real third-party systems
- Malware, credential theft, stealth/evasion instructions
- Exploit development targeting external victims

## Review process
- Keep content defensive and scope-limited.
- Add or update links/TOCs when adding files.
- Provide evidence artifacts guidance for labs.
- Ensure unit tests pass for code changes.

## Code style
- Include tests in `tests/` for Python changes.
- Keep CLI behavior stable.
