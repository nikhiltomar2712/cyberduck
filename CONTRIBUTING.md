# Contributing

Contributions should support ethical, defensive cybersecurity education.

## Guidelines

- Do not add exploit instructions for real third-party targets.
- Do not add malware, credential theft, or evasion content.
- Prefer clear explanations, checklists, and safe local labs.
- Keep examples beginner-friendly and legally safe.

## Commit style

Use short, descriptive commits, for example:

```text
Add web security checklist
Improve incident response template
Fix glossary typo
```
